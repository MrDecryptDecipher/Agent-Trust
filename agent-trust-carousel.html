Token Security

Scoped JWT Token Structure

{
  "jti": "unique-tracking-id",
  "iss": "agent-fingerprint-hex",
  "aud": "target-agent-fingerprint",
  "iat": 1735000000,
  "exp": 1735003600,
  "scopes": ["READ", "ANALYZE"],
  "task": "research_query",
  "consent_chain_id": "chain-uuid"
}
sig = Ed25519.sign( header + "." + payload, private_key )
  • jti — Unique nonce prevents replay attacks
  • iss / aud — Both issuer and audience are cryptographic fingerprints
  • scopes — Capability matrix: only listed actions permitted
  • exp — Hard expiry; no refresh tokens in zero-trust
  • consent_chain_id — Traces back to original human authorization
  • Ed25519 sig — 64-byte signature on entire payload

Graph Theory

Directed Acyclic Trust Graph

Graph Properties

  • Each edge carries a trust level enum: FULL / DELEGATE / READ / NONE
  • Edges are unidirectional — A trusts B does NOT imply B trusts A
  • Cycle detection runs on every new edge insertion
  • Root planner always at depth 0; max configurable depth enforced
  • Edge metadata includes creation timestamp + JWT scope constraints
max_delegation_depth = config.MAX_DEPTH (default: 5)

Reputation System

Bayesian Reputation Score Formula

Every agent carries a continuous [0, 1] score computed as a weighted Bayesian aggregate of three behavioral signals.

R(a) = w₁ · S(a) + w₂ · P(a) − w₃ · V(a)
Symbol Meaning Default Weight
S(a) Task success rate over sliding window w₁ = 0.50
P(a) Latency performance score (normalized) w₂ = 0.30
V(a) Policy violation deduction factor w₃ = 0.20
R(a) < 0.30 → PROVISIONAL │ R(a) < 0.15 → QUARANTINE

Merkle Tree Transaction Ledger

Tamper-Proof Guarantees

  • Change any leaf → root hash changes → detected instantly
  • O(log n) proof verification — efficient at scale
  • Root hash published to all nodes every 60s window

GDPR / SOC2 Compliance

Delegated Consent Chain

Every action traces back to the original human authorization. Each delegation hop attaches the parent consent_chain_id — creating an immutable provenance trail.

●
Human User → Root Orchestrator
Initial authorization issued. consent_chain_id = "chain-root-uuid". Full task intent scoped.
●
Root → Research Agent
Delegation: READ scope only. Agent-Trust verifies graph path & depth ≤ MAX_DEPTH.
●
Research → Parser Agent
Sub-delegation: PARSE scope. Scoped JWT TTL checked. Original chain ID preserved.
●
Parser → Audit Logger
LOG scope only. Policy engine validates original intent matches action profile.
consent_chain: Human → Root → Research → Parser → Audit [READ → PARSE → LOG]

Anomaly Detection

East-West Traffic Heuristics Engine

Three independent signal streams feed an ML ensemble. Each stream catches a different attack class.

📦 Packet Size (P)

Detects data exfiltration. Large payloads beyond agent role profile threshold trigger alert.

P_score = bytes / role_limit

⏱️ Latency (L)

Detects slow loris / timing attacks. Latency z-score beyond σ=3 from baseline flagged.

z = (t − μ) / σ > 3.0

📈 Velocity (V)

Detects DDoS / fanout attacks. Request rate per agent per window compared against role quota.

rate = req_count / window_s

Attack Vector

The Confused Deputy Attack

A classic privilege escalation where an infiltrator tricks a trusted authority into performing an action on its behalf against the victim Vault.

How Agent-Trust Blocks It

  • JWT scope is bound to both issuer AND audience fingerprint — a token for Agent A cannot be used by Agent B
  • Each token's aud must match the exact cryptographic fingerprint of the receiving agent
  • No ambient authority — zero implicit permissions from being in the same swarm
  • Action scope in token is the maximum — Vault enforces further at application layer
token.aud ≠ receiver.fingerprint → REJECT

Security Philosophy

Zero-Trust Architecture Pillars

Agent-Trust implements all five NIST Zero-Trust principles adapted for multi-agent AI systems.

Never Trust

Every request verified regardless of source location. Being inside the swarm grants zero implicit trust.

Always Verify

Signature + reputation + graph edge + scope — all four checks on every single request, no exceptions.

Least Privilege

Scoped JWTs carry only the minimum permissions needed for the specific task — not role-based blankets.

Assume Breach

Automatic blast radius minimization. Any compromise triggers isolation before propagation.

Inspect & Log

Every east-west packet inspected. Immutable Merkle ledger logs every interaction permanently.

Continuous Auth

Reputation recalculated after every interaction. Old good behavior doesn't protect against new bad behavior.

// useTrustData.js
const useTrustData = () => {
  const [state, setState] = useState(null);
  
  useEffect(() => {
    const poll = async () => {
      const res = await fetch(
        'http://localhost:8730/dashboard/metrics'
      );
      const data = await res.json();
      setState(data);
    };
    
    poll(); // immediate
    const id = setInterval(poll, 5000);
    return () => clearInterval(id);
  }, []);
  
  return state;
};
